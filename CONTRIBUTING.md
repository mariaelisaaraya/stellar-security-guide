# Contributing

Thanks for helping make Stellar safer. This guide is community-driven and every
contribution counts.

## Ways to contribute

- **Report something** — outdated command, broken link, a claim that no longer
  matches the official Stellar docs. Open an issue.
- **Share experience** — you run a node or shipped a contract? Add the lessons
  you learned the hard way.
- **Add tooling** — a new analyzer, monitor, or pattern worth documenting.
- **Translate** — help keep the Spanish and English versions in sync.

## How to submit

1. Fork the repo and create a branch (`fix/ttl-section`, `add/oracle-checks`).
2. Make your change. Keep it practical and cite a source where it matters.
3. Open a pull request. The PR template will guide you.

## Ground rules

- **Cite primary sources.** When in doubt, the official
  [Stellar docs](https://developers.stellar.org) and
  [`stellar-dev-skill`](https://github.com/stellar/stellar-dev-skill) win.
- **No copy-paste from copyrighted material.** Write in your own words.
- **Be kind.** Reviews are about the code, not the person.
- **Test what you can.** If you add a command or config, run it first.

## Scope

This repo covers Soroban contract security and Stellar node operation. Frontend
and general web security are out of scope (link to good external resources
instead of duplicating them).
